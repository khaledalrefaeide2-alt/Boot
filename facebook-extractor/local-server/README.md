# قاعدة البيانات المحلية 📦

خادم صغير يخزّن كل المنشورات المستخرجة والمرصودة في ملف **SQLite** دائم على جهازك (`fbx-posts.db` يُنشأ في هذا المجلد تلقائياً). بدون أي مكتبات خارجية — ملف واحد فقط.

## المتطلبات

- [Node.js](https://nodejs.org) الإصدار 22.5 أو أحدث (تثبيت مرة واحدة فقط).

## التشغيل

| النظام | الطريقة |
|---|---|
| **ويندوز** | اضغط مرتين على `تشغيل-قاعدة-البيانات.bat` |
| **ماك / لينكس** | نفّذ `bash run.sh` أو `node server.js` |

اترك النافذة مفتوحة — الخادم يعمل على `http://localhost:3300`.

## الربط بالموقع

1. افتح الموقع (الصفحة الرئيسية أو صفحة الرصد).
2. في قسم **«قاعدة البيانات المحلية»** اضغط **«🔌 فحص الاتصال»**.
3. عند ظهور 🟢 يبدأ الحفظ التلقائي:
   - الصفحة الرئيسية: كل استخراج يُحفظ تلقائياً، وزر «💾 حفظ في قاعدة البيانات» للحفظ اليدوي.
   - صفحة الرصد: كل منشور جديد ترصده الدورات يُحفظ تلقائياً.

التكرار مستحيل — كل منشور يُخزَّن مرة واحدة (حسب رابطه) وتُحدَّث أرقام تفاعلاته فقط.

## واجهة الـ API

| الطلب | الوظيفة |
|---|---|
| `GET /api/health` | حالة الخادم وعدد المنشورات |
| `POST /api/posts` | حفظ منشورات `{posts:[...]}` |
| `GET /api/posts?limit=&offset=&q=` | جلب المنشورات من الأحدث للأقدم مع بحث اختياري |
| `GET /api/stats` | إحصائيات إجمالية |
| `DELETE /api/posts` | مسح كل المنشورات |

ملف قاعدة البيانات `fbx-posts.db` يمكن فتحه بأي أداة SQLite (مثل DB Browser for SQLite) للاستعلامات المتقدمة.
